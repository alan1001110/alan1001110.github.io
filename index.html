Choosing an Editor

After trying a few text editors and a couple IDE's, I've recently settled in with Brackets. It's perfect for the web stuff I'm doing now. First, a little about what else I tried.

I started out with Notepad++ (free), but I wanted a little more.

I tried Netbeans and Cloud 9's IDE's, but they're so slow in comparison to a text-editor, and I found that frustrating.

I loved the speed of Sublime and its default Monokai theme, but my brief attempt at trying to install custom packages was frustrating. I also didn't want to pay for it while still trying it out, and not making an income at coding. The constant pop-ups asking for money were annoying.

I read about Atom, but didn't feel the need to try it out.

Visual Studio Code was pretty good, but I didn't like it quite as much as Brackets.

Which brings me what I like about Brackets. It feels slick and professional. It's fast. It's open-source and free. It has a built-in Live Preview feature that works with the Chrome browser and updates as you type. (Live Preview can be used for either static sites or with a back-end server.) Brackets is also super-easy to customize with third-party linters, debuggers, themes, and more if you want to replace or build on what it already comes with. Just click the Extension Manager that looks like a Lego block in the upper right corner, select an extension, click install, and Bob's your uncle.

My key customizations: First, I chose the Monokai theme and bumped the font size up a little to make it look a lot like Sublime. Second, I installed the Brackets Git extension by Martin Zegora. It is surprisingly full-featured and a pleasure to use.

I've also got Brackets-InteractiveLinter installed, plus a couple other extensions.

For anyone starting out in web development with Javascript/Node, I highly recommend Brackets (especially with my key customizations).
